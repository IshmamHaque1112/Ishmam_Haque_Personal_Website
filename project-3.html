Project 3: Genetic Knapsack Problems

Designed three Java programs solving the knapsack problem using brute force, random search, and genetic algorithm approaches.

Putting it in the Bag

In my genetic algorithms class, I was assigned to handle the knapsack problem to handle sorting. The knapsack problem is about how many items from a set one can fit into a bag with a maximum value while taking into account the maximum weight space of the bag. I was asked to program a brute force solution, a solution that generates random samples, and a genetic algorithm solution. I managed to complete them in a timely manner.

Part 1: Every solution possible

In order to run a brute force solution, I had to first generate every solution possible. I did so by generating all permutations of the knapsack problem using a function. Then I removed those over the weight limit, and found the solution with the max value.

Part 2: Randomization

This approach relied on the greedy algorithm approach. I generated 100 generations, 1000 members each. Throughout this process, it tracked which was the best cost solution at the time, and ultimately presented a final combination when the program came to an end.

Part 3: Genetics

This was the solution that allowed me to demonstrate my understanding of genetic algorithms. In genetic algorithms, one generates a series of solutions and breeds the best solutions over many generations to get the best solutions possible. Likewise, I built on my random generation program and created something much more layered. Starting with one generation of 1000 solutions, I bred the next generation by selecting the best using a criteria from the current generation and breeding them. This process continued over a hundred generations until finally culminating in the best generation with optimal solutions.
